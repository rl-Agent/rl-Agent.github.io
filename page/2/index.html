React 元素是不可变对象。一旦被创建，你就无法更改它的子元素或者属性。一个元素就像电影的单帧：它代表了某个特定时刻的 UI。

尽管每一秒我们都会新建一个描述整个 UI 树的元素，React DOM 只会更新实际改变了的内容，也就是例子中的文本节点。

JS同步和异步

发表于 2019-06-01 | 分类于 js

1、单线程

  • 什么是单线程

JavaScript是一门单线程语言，单线程意味着若在同一时间有多个任务需要执行，那必须得排队执行。

在处理简单数据时响应固然快，而当foo1函数在执行文件读取或ajax等需要处理较多的数据时，都会耗费一定的时间。

  • 为什么是单线程
    JavaScripts是浏览器的脚本语言，我们对DOM进行各种操作与用户交互，现在若有一个线程要对一个dom节点增加内容，而另一个线程要对这个dom节点执行删减命令，就像现在只有一个魔方，小明想玩这个魔方，同时小亮也想玩这个魔方，两人又不能同时玩这个魔方，那不得干一架？为了避免这种情况，就需要一人先玩然后另一人再玩。这就是同步，所以JavaScript是单线程的
  • 解决单线程带来的问题
    JavaScript在被设计的时候，设计者已经提供出了方法。主线程不必等待大量的数据请求，可以先挂起等待中的任务，然后直接执行后面排着的任务，等到数据全部获取后，在回过去执行先前的任务，这样便分成了同步任务和异步任务

    2、同步任务和异步任务

  • 同步任务
    当在主线程上执行排队任务时，必须等前一个任务执行完毕才能执行下一个任务。例：页面打开时元素的渲染
  • 异步任务
    不进入主线程，而是另外排入任务队列中，当该异步任务可执行时，便进入主线程执行。例：页面打开是图片和音频的加载
  • 为什么会有同步和异步任务
    难道我们必须得等foo1函数执行完毕才能再执行foo2函数吗？显然不用，拿ajax为例，当用户向后台获取大量数据时，必须等到所有数据获取完毕才能继续下一步操作，这样的话就产生了不必要的用户等待时间。
  • JavaScript的异步机制的四个步骤
  1. 所有同步任务都在主线程上执行，行成一个执行栈
  2. 主线程之外，还存在一个任务队列，只要异步任务有了结果，就会在任务队列中放置一个事件
  3. 一旦执行栈中的所有同步任务执行完毕，系统就会读取任务队列，看看里面还有哪些事件，那些对应的异步任务，于是结束等待状态，进入执行栈，开始执行
  4. 主线程不断的重复上面的第三步
